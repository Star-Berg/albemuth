# 训练任务

## 1. 使用 DAC 通道 A 发生一个 100Hz 的正弦波

使用 DAC 通道 A 发生一个 100Hz 的正弦波，更新频率 20kHz，峰峰值为 1.65V，偏置为 1.65V。

> 注：发生正弦波需要使用 `ctl_sin()` 函数。

### 分析

由 **5.1.2 DAC 输出** 部分可知，其需要在 `output_callback` 中完成，可定位到 `xplt.ctl_interface.h`，垂直分辨率为 12bit（4096），格式为：

```c
DAC_setShadowValue(IRIS_DACA_BASE,
                   (uint16_t)(adc_v_out.control_port.value * 2048.0f + 2048.0f));
```

其中，`IRIS_DACA_BASE` 代表 DAC 的 A 通道，第二个参数包括三个部分，要注意其输出的为标幺值，因此需要进行转换，因为输入为 [-1, 1]，即 `* 2048.0f` 代表等比例缩放（恰好覆盖 4096），最后 `+ 2048` 的作用相当于给 DAC 加了一个中点偏置。

了解了函数的具体形式，我们还需知道电源电压/DAC 满量程是 3.3V，但题目要求输出的正弦波不是满量程正弦，而是峰峰值 1.65V、偏置 1.65V，因此实际 DAC 输出电压应该是：

$$
V_{out} = 1.65 + 0.825\sin\theta
$$

换成 DAC 数字量：

$$
DAC_{code} = \frac{1.65 + 0.825\sin\omega t}{3.3}\times4096 = 2048 + 1024\sin\omega t
$$

因此实际代码为：

```c
GMP_STATIC_INLINE void ctl_output_callback(void)
{
    static uint32_t tick = 0;

    tick += 1;

    ctrl_gt output_signal = (tick % 20000) / 20000.0f * 100;
    //1.65V+1.65/2 sin(100*2/pi * t)
    DAC_setShadowValue(IRIS_DACA_BASE, ctl_sin(output_signal) * 1024 + 2048);
}
```

## 2. ADC 采集正弦波并标幺化

### 题目要求

使用 ADC 采集 DAC 输出的正弦波，并且减去 `1.65 V` 偏置，以 `1.65 V / 2` 为单位 `1` 进行标幺化。

题目提示正弦波的标幺化可以使用 `adc_channel_t` 类型提供的功能，参考头文件为：

```text
gmp_pro\ctl\component\interface\adc_channel.h
```

---

### 2.1 `ctl_init_adc_channel()` 函数说明

`adc_channel_t` 类型在使用前需要先进行初始化，其初始化函数形式为：

```c
void ctl_init_adc_channel(
    adc_channel_t* adc_obj,
    ctrl_gt gain,
    ctrl_gt bias,
    fast_gt resolution,
    fast_gt iqn
);
```

各参数含义如下：

| 参数 | 含义 |
|---|---|
| `adc_obj` | 需要初始化的 ADC 通道对象 |
| `gain` | ADC 标幺化时使用的增益 |
| `bias` | ADC 标幺化时使用的偏置 |
| `resolution` | ADC 分辨率，本题为 12 bit |
| `iqn` | 内部定点数格式的小数位数，本实验中使用 24 |

---

### 2.2 为什么初始化参数写成 `4.0f, 0.5f, 12, 24`

本题中 ADC 和 DAC 的参考电压均为 `3.3 V`，ADC 为 `12 bit`，因此 ADC 原始采样值范围为：

```text
0 ~ 4095
```

题目要求采集的正弦波为：

```text
峰峰值：1.65 V
偏置：1.65 V
```

因此正弦波实际电压范围为：

```text
0.825 V ~ 2.475 V
```

将其除以 ADC 参考电压 `3.3 V` 后，得到归一化范围：

```text
0.25 ~ 0.75
```

其中：

```text
0.825 / 3.3 = 0.25
1.65  / 3.3 = 0.50
2.475 / 3.3 = 0.75
```

所以 ADC 归一化后的信号中心为 `0.5`，幅值为 `0.25`。

题目要求减去 `1.65 V` 偏置，并以 `0.825 V` 为单位 `1`，也就是希望得到范围约为 `[-1, 1]` 的标幺正弦波。

因此标幺化关系为：

```text
x_pu = (x_adc_norm - 0.5) / 0.25
```

也可以写成：

```text
x_pu = 4.0 * (x_adc_norm - 0.5)
```

所以初始化时：

```c
gain = 4.0f;
bias = 0.5f;
resolution = 12;
iqn = 24;
```

对应代码为：

```c
ctl_init_adc_channel(&input_wave_adc, 4.0f, 0.5f, 12, 24);
```

---

### 2.3 验证标幺化关系

| ADC 输入电压 | ADC 归一化值 | 标幺结果 |
|---|---:|---:|
| `0.825 V` | `0.25` | `4 × (0.25 - 0.5) = -1` |
| `1.65 V` | `0.50` | `4 × (0.50 - 0.5) = 0` |
| `2.475 V` | `0.75` | `4 × (0.75 - 0.5) = 1` |

因此，使用：

```c
ctl_init_adc_channel(&input_wave_adc, 4.0f, 0.5f, 12, 24);
```

可以将 ADC 采集到的 `0.825 V ~ 2.475 V` 正弦波标幺化为 `-1 ~ 1` 的正弦波。

---

### 2.4 变量定义

在 `.c` 文件的全局位置定义 ADC 通道对象：

```c
adc_channel_t input_wave_adc;
```

该变量用于保存 ADC 通道的配置、状态以及标幺化后的输出结果。

---

### 2.5 初始化代码

初始化函数中调用 `ctl_init_adc_channel()`：

```c
void setup_peripheral(void)
{
    .
    .
    .
    ctl_init_adc_channel(&input_wave_adc, 4.0f, 0.5f, 12, 24);
    .
    .
    .
}
```

其中：

```text
4.0f：标幺化增益
0.5f：归一化后的中点偏置
12：ADC 分辨率为 12 bit
24：内部 IQ 格式小数位数
```

---

### 2.6 ADC 采样与标幺化

ADC 输入应在 `input_callback` 中完成。

ADC 通道配置为：

```c
#define ADC_CH1 ADC_SOC_NUMBER0
#define ADC_CH1_FORCE ADC_FORCE_SOC0
#define ADC_CH1_ADC_BASE ADCA_BASE
#define ADC_CH1_RESULT_BASE ADCARESULT_BASE
#define ADC_CH1_SAMPLE_WINDOW 150
#define ADC_CH1_TRIGGER_SOURCE ADC_TRIGGER_EPWM1_SOCA
#define ADC_CH1_CHANNEL ADC_CH_ADCIN6
```

则定义：

```c
#define INPUT_WAVE_RESULT_REG    ADCARESULT_BASE
#define INPUT_WAVE               ADC_SOC_NUMBER0
```

对应代码为：

```c
GMP_STATIC_INLINE void ctl_input_callback(void)
{
    ctl_step_adc_channel(&input_wave_adc, ADC_readResult(INPUT_WAVE_RESULT_REG, INPUT_WAVE));
}
```

其中：

```c
ADC_readResult(INPUT_WAVE_RESULT_REG, INPUT_WAVE);
```

用于读取 ADC 原始采样值，范围为 `0 ~ 4095`。

```c
ctl_step_adc_channel(&input_wave_adc, ADC_readResult(...))
```

用于将 ADC 原始值送入 `adc_channel_t`，并得到标幺化后的结果。

## 3. 使用超前补偿器进行超前补偿

使用超前补偿器对于标幺后的信号做超前补偿，超前补偿器的补偿频率设置为 100Hz，超前角度设置为 45°（π/4），增益为 1。

> 注：超前补偿器可以使用 `ctl_lead_t` 模块。
>
> 参考头文件：`gmp_pro\ctl\component\intrinsic\discrete\lead_lag.h`

## 5. 通过 PWM 输出补偿后的波形

> 注：PWM 调制器的输出可以使用 `pwm_channel_t` 模块。
