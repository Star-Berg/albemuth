# 电设项目仓库

本仓库用于存放电子设计大赛相关资料、代码、文档和调试记录，便于团队成员协作开发、版本管理和后续整理。

## 项目简介

本项目面向电子设计大赛，主要围绕硬件设计、嵌入式程序开发、控制算法实现以及系统调试展开。仓库将持续记录项目从方案设计、代码编写、功能验证到最终作品完善的全过程。

## 仓库内容

```text
.
├── docs/          # 项目文档、题目解析、设计说明
├── hardware/      # 原理图、PCB、器件资料等硬件相关文件
├── firmware/      # 单片机、DSP、控制器等嵌入式代码
├── software/      # 上位机、仿真、辅助脚本等软件代码
├── test/          # 测试程序、实验数据、调试记录
└── README.md      # 仓库说明文件
```

## 当前目标

- 完成比赛题目分析与技术方案设计
- 搭建基本硬件和软件框架
- 实现核心控制功能
- 完成系统联调与性能优化
- 整理项目文档、代码说明和最终展示材料

## 开发规范

1. 每次修改后及时提交，提交信息应简洁说明本次改动内容。
2. 代码文件应尽量添加必要注释，方便后续维护。
3. 文档优先使用 Markdown 格式，便于在 GitHub 上直接预览。
4. 重要实验结果、调试问题和解决方法应记录在 `docs/` 或 `test/` 文件夹中。
5. 不建议直接提交临时文件、编译中间文件或无关的大文件。
